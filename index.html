1

Princípios

Antes das regras, a filosofia. Estes são os três princípios que explicam por que as telas do Contele Teams se parecem entre si.

A interface é monocromática no azul

O Contele Teams tem uma única cor de destaque na interface: o azul #4a58ff. Botões de ação, estados ativos, ícones de filtro e links usam esse azul — e só ele. Não existem botões verdes, laranjas ou de outras cores no chrome da interface.

Regra-mãe: "cor de UI" ≠ "cor de dado". A interface (botões, menus, abas) é monocromática no azul. Cores como verde, ciano, laranja ou vermelho só aparecem como codificação de informação dentro de gráficos e indicadores — nunca como decoração ou no chrome da tela.

Componentes vêm do shadcn/ui

A biblioteca base de componentes é o shadcn/ui (ui.shadcn.com). Ela é o ponto de partida — as regras deste guia definem como o Contele Teams customiza e usa esses componentes.

Para quem é este guia

Para o designer de produto do Contele Teams e para qualquer pessoa que entre no time e precise criar ou revisar telas web sem ter o contexto histórico na cabeça.

O duplo papel do vermelho

O vermelho é a única cor com dois papéis. O designer não pode confundi-los:

PapelOnde apareceExemplo
Vermelho de UIApenas ações destrutivasBotão/ação "Excluir", "Remover"
Vermelho de dadoIndicador negativo em gráfico/KPITendência de queda ↓
Como dado (queda), o vermelho é permitido livremente. Como UI, o vermelho é exclusivo de ações destrutivas — nunca decorativo, nunca em botões comuns.

Os hex do vermelho destrutivo e do cinza neutro ainda serão confirmados a partir de uma tela. Os valores exibidos acima são aproximações de referência.

A única exceção: o verde comercial

Existe uma exceção à regra "azul é a única cor de UI": botões de incentivo comercial para contratar ou expandir licenças são verdes (ex.: "Contratar mais licenças").

Escopo estrito. O verde só é permitido em CTAs de contratar/expandir licenças. Não vale para "salvar", "confirmar", "concluir" ou qualquer outra ação positiva — essas continuam azuis.

6

Iconografia

Ícones consistentes em toda a interface.

  • Estilo linha/outline, traço fino e uniforme.
  • Nos menus, o ícone fica à esquerda do rótulo; em cards de KPI, no canto superior direito.
  • Na barra de ações contextual, ícone outline acompanha o texto da ação.
A biblioteca de ícones usada no produto ainda será confirmada (ex.: Lucide, que é o padrão do shadcn/ui). Este documento usa Font Awesome apenas para ilustração.

8

Boas práticas

Os do's & don'ts que resumem as regras em decisões rápidas.

Faça

  • Filtros à esquerda, ações à direita em toda barra de ação.
  • Use o azul #4a58ff como única cor de UI.
  • Deixe ações secundárias visíveis numa barra contextual com ícone + texto.
  • Reserve o vermelho de UI só para ações destrutivas.
  • Use o botão primário (azul) para a ação principal e o secundário (cinza + texto azul) para as demais.
  • Use o verde apenas em CTA de contratar/expandir licenças — a única exceção comercial.

Não faça

  • Não use menu kebab (três pontinhos na vertical).
  • Não use verde, laranja ou outras cores como cor de UI — só como cor de dado em gráficos.
  • Não use vermelho decorativo ou em botões comuns.
  • Não invente um estilo de seleção diferente do definido para cada menu.
9

Checklist de conformidade próximo passo

Esta seção vai consolidar os critérios verificáveis de uma tela — e será a base do prompt avaliador que vamos montar para checar telas automaticamente.

  • Cor de destaque é o azul #4a58ff? Sem outras cores de UI?
  • Filtros à esquerda e ações à direita?
  • Botão primário azul / secundário cinza usados corretamente?
  • Vermelho apenas em ação destrutiva?
  • Sem menu kebab?
  • Estados de seleção corretos por menu?
  • Ícones outline e consistentes?
  • Tipografia Nunito Sans?
Quando o conteúdo das seções estiver maduro, transformamos este checklist no prompt de avaliação de telas.
